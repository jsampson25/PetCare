## UX documentation rules

- Organize the product around user goals and workflows, not database tables.
- Use the same names for the same concepts across surfaces unless the audience requires a clearer customer-facing term.
- Show only navigation destinations the user is authorized to access.
- Hiding a control is not authorization; server-side enforcement remains mandatory.
- Preserve context when a user moves between related screens.
- Prefer updating a shared journey or pattern over documenting inconsistent screen-specific behavior.
- Do not create detailed screens for deferred capabilities before their product scope is accepted.
